## Overview

SupaScan supports three output formats to suit different use cases:

1. **Console** (default) - Human-readable formatted output
2. **JSON** - Machine-readable structured data
3. **HTML** - Interactive web-based report

## Console Output (Default)

The default console output is designed for human readability with colors, formatting, and visual indicators.

### Basic Usage

```bash
supascan --url <url> --key <key>
```

### Analyze Command Output

**Structure:**

```
============================================================
  SUPABASE DATABASE ANALYSIS
============================================================

TARGET SUMMARY
--------------------
Domain: your-project.supabase.co
Service: supabase
Project ID: your-project
Title: PostgREST API
Version: 11.2.2

JWT TOKEN INFO
--------------------
Issuer: https://your-project.supabase.co/auth/v1
Audience: authenticated
Role: anon
Expires: 2030-01-01T00:00:00.000Z
Issued: 2024-01-01T00:00:00.000Z

DATABASE ANALYSIS
--------------------
Schemas discovered: 2

Schema: public

Tables: 5
  3 exposed | 1 empty/protected | 1 denied

  [+] users (~42 rows exposed)
  [+] posts (~150 rows exposed)
  [+] comments (~89 rows exposed)
  [-] profiles (0 rows - empty or RLS)
  [X] admin_settings (access denied)

RPCs: 2
  * get_user_posts
    - user_id: uuid (required)
  * search_posts
    - query: text (required)
    - limit: int4 (optional)

```

**Visual Indicators:**

- **[+]** Green - Table is readable with data exposed
- **[-]** Yellow - Table returns 0 rows (empty or RLS protected)
- **[X]** Red - Access denied

## JSON Output

JSON output is ideal for programmatic access, CI/CD integration, or piping to other tools.

### Basic Usage

```bash
supascan --url <url> --key <key> --json
```

All output goes to **stdout**, while log messages go to **stderr**.

### Processing JSON Output

**Save to file:**
```bash
supascan --url <url> --key <key> --json > analysis.json
```

**Use with jq:**
```bash
supascan --url <url> --key <key> --json | jq '.schemas[]'
```

**Count exposed tables:**
```bash
supascan --url <url> --key <key> --json | \
  jq '[.schemaDetails.public.tableAccess | to_entries[] | select(.value.status == "readable")] | length'
```

**Extract all RPC names:**
```bash
supascan --url <url> --key <key> --json | \
  jq -r '.schemaDetails.public.rpcFunctions[].name'
```

## HTML Output

The HTML output generates an interactive web-based report with embedded analysis capabilities.

### Basic Usage

```bash
supascan --url <url> --key <key> --html
```

**Output:**
```
✓ HTML report generated: /tmp/supascan-report-abc123.html
```

The report automatically opens in your default browser.

### Features

**Interactive UI:**
- Expandable sections for schemas, tables, and RPCs
- Color-coded access indicators
- Sortable and filterable results
- Real-time search

**Embedded Analysis:**
- Credentials are embedded in the HTML file
- Can re-run analysis from the browser
- No server required

**Sharing:**
- Single HTML file
- No dependencies
- Works offline

### Security Considerations

<Warning>
The HTML report contains your Supabase credentials encoded in the file:

```html
<script>window.__SUPASCAN_CONFIG__ = "eyJ1cmwiOiJodHRwczovL...";</script>
```

**Important:**
- Keep the HTML file secure
- Don't commit to version control
- Delete after viewing
- Don't share publicly
</Warning>

### HTML Report Location

Reports are saved to temporary directory:

**Format:** `/tmp/supascan-report-{random}.html`

**Find all reports:**
```bash
ls /tmp/supascan-report-*.html
```

**Clean up old reports:**
```bash
rm /tmp/supascan-report-*.html
```

## Choosing the Right Format

### Use Console Output When:

- Performing quick manual security audits
- Exploring a database interactively
- Need immediate visual feedback
- Working in terminal environment

### Use JSON Output When:

- Integrating with CI/CD pipelines
- Processing results programmatically
- Storing results for later analysis
- Building automation tools
- Comparing multiple scans

### Use HTML Output When:

- Sharing results with team members
- Need interactive exploration
- Want to save a visual report
- Presenting findings in meetings
- Need to re-run analysis later

## Output Restrictions

### Conflicting Options

You cannot use `--json` and `--html` together:

```bash
supascan --url <url> --key <key> --json --html
```

**Error:**
```
✖ Cannot use --json and --html together. Please choose one.
```

### Output Streams

**stdout:** Analysis results (JSON or console formatted)

**stderr:** Log messages (info, success, error, warning)

**Benefits:**
- Clean separation of data and logs
- Easy piping: `supascan ... --json | jq`
- Logs visible even when redirecting output

## Examples

### Save Console Output

```bash
supascan --url <url> --key <key> 2>&1 | tee scan.log
```

### JSON Only (No Logs)

```bash
supascan --url <url> --key <key> --json 2>/dev/null
```

### Compare Two Scans

```bash
supascan --url <url> --key <key> --json > scan1.json
# ... make changes ...
supascan --url <url> --key <key> --json > scan2.json
diff <(jq -S . scan1.json) <(jq -S . scan2.json)
```

### CI/CD Integration

```bash
#!/bin/bash
set -e

# Run scan and save results
supascan --url "$SUPABASE_URL" --key "$SUPABASE_KEY" --json > scan.json

# Check for exposed tables
EXPOSED=$(jq '[.schemaDetails.public.tableAccess | to_entries[] | select(.value.status == "readable")] | length' scan.json)

if [ "$EXPOSED" -gt 0 ]; then
  echo "Warning: $EXPOSED tables are exposed!"
  exit 1
fi
```
