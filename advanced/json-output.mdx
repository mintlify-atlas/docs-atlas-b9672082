## Overview

SupaScan can output results as JSON using the `--json` flag. This enables:

- Integration with security automation tools
- Scripted analysis and reporting
- Long-term storage and trending
- Custom alerting and notifications
- Data processing with tools like `jq`

## Basic Usage

Add the `--json` flag to any SupaScan command:

```bash
supascan --url https://xyz.supabase.co --key eyJhbG... --json
```

Output is printed to stdout as formatted JSON:

```json
{
  "schemas": ["public", "auth"],
  "schemaDetails": {
    "public": {
      "tables": ["users", "posts"],
      "rpcs": ["get_user_stats"],
      "tableAccess": {
        "users": {
          "status": "readable",
          "accessible": true,
          "hasData": true,
          "rowCount": 1420
        }
      }
    }
  },
  "summary": {
    "domain": "xyz.supabase.co",
    "jwtInfo": {
      "role": "anon",
      "iss": "https://xyz.supabase.co/auth/v1"
    }
  }
}
```

## Redirecting to Files

Save JSON output to a file:

```bash
supascan --url https://xyz.supabase.co --key eyJhbG... --json > scan-results.json
```

<Info>
SupaScan writes JSON to **stdout** and logs/errors to **stderr**, so you can safely redirect output without capturing error messages.
</Info>

## Processing with jq

Use `jq` to query and transform JSON output:

<AccordionGroup>
  <Accordion title="Count Exposed Tables">
    ```bash
    supascan --url https://xyz.supabase.co --key eyJhbG... --json | \
      jq '[.schemaDetails[].tableAccess[] | select(.status == "readable")] | length'
    ```
    
    Output: `3` (number of readable tables)
  </Accordion>

  <Accordion title="List Table Names by Status">
    ```bash
    supascan --url https://xyz.supabase.co --key eyJhbG... --json | \
      jq '.schemaDetails.public.tableAccess | to_entries | map(select(.value.status == "readable") | .key)'
    ```
    
    Output: `["users", "posts", "comments"]`
  </Accordion>

  <Accordion title="Extract JWT Role">
    ```bash
    supascan --url https://xyz.supabase.co --key eyJhbG... --json | \
      jq -r '.summary.jwtInfo.role'
    ```
    
    Output: `anon`
  </Accordion>

  <Accordion title="Get Total Row Counts">
    ```bash
    supascan --url https://xyz.supabase.co --key eyJhbG... --json | \
      jq '[.schemaDetails[].tableAccess[] | select(.rowCount != null) | .rowCount] | add'
    ```
    
    Output: `5512` (total rows across all readable tables)
  </Accordion>

  <Accordion title="Format Exposed Tables Report">
    ```bash
    supascan --url https://xyz.supabase.co --key eyJhbG... --json | \
      jq -r '.schemaDetails | to_entries[] | .key as $schema | .value.tableAccess | to_entries[] | select(.value.status == "readable") | "\($schema).\(.key): \(.value.rowCount) rows"'
    ```
    
    Output:
    ```
    public.users: 1420 rows
    public.posts: 892 rows
    public.comments: 3200 rows
    ```
  </Accordion>
</AccordionGroup>

## Integration Examples

<Tabs>
  <Tab title="CI/CD Pipeline">
    Fail builds if exposed tables are found:
    
    ```bash .github/workflows/security-scan.yml
    name: Security Scan
    on: [push]
    
    jobs:
      scan:
        runs-on: ubuntu-latest
        steps:
          - name: Run SupaScan
            run: |
              npx supascan --url ${{ secrets.SUPABASE_URL }} --key ${{ secrets.SUPABASE_KEY }} --json > results.json
          
          - name: Check for exposed tables
            run: |
              exposed=$(jq '[.schemaDetails[].tableAccess[] | select(.status == "readable")] | length' results.json)
              if [ "$exposed" -gt 0 ]; then
                echo "ERROR: Found $exposed exposed tables!"
                jq -r '.schemaDetails[].tableAccess | to_entries[] | select(.value.status == "readable") | .key' results.json
                exit 1
              fi
    ```
  </Tab>
  
  <Tab title="Slack Notifications">
    Send alerts to Slack when issues are found:
    
    ```bash notify.sh
    #!/bin/bash
    
    # Run scan
    supascan --url $SUPABASE_URL --key $SUPABASE_KEY --json > scan.json
    
    # Count exposed tables
    exposed=$(jq '[.schemaDetails[].tableAccess[] | select(.status == "readable")] | length' scan.json)
    
    if [ "$exposed" -gt 0 ]; then
      # Build Slack message
      tables=$(jq -r '.schemaDetails[].tableAccess | to_entries[] | select(.value.status == "readable") | .key' scan.json | paste -sd "," -)
      
      curl -X POST $SLACK_WEBHOOK_URL \
        -H 'Content-Type: application/json' \
        -d "{
          \"text\": \"⚠️ SupaScan Alert: $exposed exposed tables found\",
          \"attachments\": [{
            \"color\": \"danger\",
            \"fields\": [{
              \"title\": \"Exposed Tables\",
              \"value\": \"$tables\"
            }]
          }]
        }"
    fi
    ```
  </Tab>
  
  <Tab title="Security Dashboard">
    Store results in a database for trending:
    
    ```python store_results.py
    import json
    import sqlite3
    from datetime import datetime
    import subprocess
    
    # Run scan
    result = subprocess.run(
        ['supascan', '--url', SUPABASE_URL, '--key', SUPABASE_KEY, '--json'],
        capture_output=True,
        text=True
    )
    
    scan_data = json.loads(result.stdout)
    
    # Store in database
    conn = sqlite3.connect('security_scans.db')
    cursor = conn.cursor()
    
    cursor.execute('''
        INSERT INTO scans (timestamp, exposed_tables, total_tables, raw_data)
        VALUES (?, ?, ?, ?)
    ''', (
        datetime.now(),
        len([t for t in scan_data['schemaDetails']['public']['tableAccess'].values() if t['status'] == 'readable']),
        len(scan_data['schemaDetails']['public']['tables']),
        json.dumps(scan_data)
    ))
    
    conn.commit()
    conn.close()
    ```
  </Tab>
  
  <Tab title="Comparison Script">
    Compare scans over time:
    
    ```bash compare.sh
    #!/bin/bash
    
    # Run scan and save
    supascan --url $SUPABASE_URL --key $SUPABASE_KEY --json > scan-$(date +%Y%m%d).json
    
    # Compare with previous scan
    if [ -f scan-previous.json ]; then
      echo "Comparing with previous scan..."
      
      # Get table lists
      current=$(jq -r '.schemaDetails.public.tableAccess | to_entries[] | select(.value.status == "readable") | .key' scan-$(date +%Y%m%d).json | sort)
      previous=$(jq -r '.schemaDetails.public.tableAccess | to_entries[] | select(.value.status == "readable") | .key' scan-previous.json | sort)
      
      # Find new exposures
      new=$(comm -13 <(echo "$previous") <(echo "$current"))
      
      if [ -n "$new" ]; then
        echo "⚠️ NEW EXPOSED TABLES:"
        echo "$new"
      else
        echo "✅ No new exposures"
      fi
    fi
    
    # Update previous
    cp scan-$(date +%Y%m%d).json scan-previous.json
    ```
  </Tab>
</Tabs>

## Output vs HTML

<Warning>
You cannot use `--json` and `--html` together. Choose one output format per run.
</Warning>

| Format | Use Case | Pros | Cons |
|--------|----------|------|------|
| **--json** | Automation, scripting, CI/CD | Machine-readable, pipeable, parseable | Not human-friendly |
| **--html** | Manual inspection, reporting | Interactive, visual, easy to explore | Not scriptable |
| **Console** (default) | Quick checks, terminal use | Human-readable, colored output | Hard to parse |
