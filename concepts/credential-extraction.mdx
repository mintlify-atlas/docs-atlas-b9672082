## Overview

Credential extraction is the first and most critical step in SupaScan's security analysis workflow. It automatically discovers Supabase API URLs and anonymous keys exposed in web applications by analyzing HTML, JavaScript files, and inline scripts.

## How It Works

SupaScan uses pattern matching and proximity algorithms to locate Supabase credentials embedded in client-side code. The extraction process follows a systematic approach:

### 1. Content Detection

When you provide a URL, SupaScan first determines the content type:

```typescript
const isHtml =
  contentType.includes("text/html") ||
  content.trim().startsWith("<!DOCTYPE") ||
  content.trim().startsWith("<html");

const isJs =
  url.endsWith(".js") ||
  contentType.includes("javascript") ||
  contentType.includes("ecmascript");
```

This detection ensures the right extraction strategy is applied based on whether you're scanning an HTML page or a JavaScript file directly.

### 2. Pattern Matching

SupaScan uses regular expressions to identify Supabase credentials. There are two primary patterns it looks for:

**URL Patterns:**
```typescript
const URL_PATTERNS = [
  /https:\/\/[a-z0-9-]+\.supabase\.co\/?/g,
  /['"`]https:\/\/[a-z0-9-]+\.supabase\.co\/?['"`]/g,
];
```

**Key Patterns (JWT tokens):**
```typescript
const KEY_PATTERNS = [
  /eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+/g,
  /['"`]eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9\.[A-Za-z0-9_-]+\.[A-Za-z0-9_-]+['"`]/g,
];
```

The JWT pattern specifically matches Supabase's token format, which always starts with `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9` (the base64-encoded JWT header).

### 3. Framework-Specific Detection

SupaScan has special handling for common Supabase client initialization patterns:

```typescript
const CREATE_BROWSER_CLIENT_PATTERN =
  /createBrowserClient\)\s*\(\s*["']([^"']+)["']\s*,\s*["']([^"']+)["']/g;
```

This pattern detects the `createBrowserClient()` function call commonly used in Next.js and other frameworks:

```javascript
// Example code that would be detected
const supabase = createBrowserClient(
  'https://project.supabase.co',
  'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...'
);
```

### 4. Proximity Algorithm

When credentials aren't found in a structured function call, SupaScan uses a proximity-based matching algorithm to pair URLs with keys:

```typescript
function findClosestPairs(
  content: string,
): Array<{ url: string; key: string; distance: number }> {
  const urlMatches = findAllMatches(content, URL_PATTERNS);
  const keyMatches = findAllMatches(content, KEY_PATTERNS);

  const pairs: Array<{ url: string; key: string; distance: number }> = [];

  for (const urlMatch of urlMatches) {
    for (const keyMatch of keyMatches) {
      const distance = Math.abs(urlMatch.index - keyMatch.index);
      pairs.push({
        url: urlMatch.text.replace(/['"`;]/g, ""),
        key: keyMatch.text.replace(/['"`;]/g, ""),
        distance,
      });
    }
  }

  return pairs.sort((a, b) => a.distance - b.distance);
}
```

This algorithm:
- Finds all URL and key matches in the content
- Calculates the character distance between each URL-key pair
- Returns pairs sorted by proximity (closest first)
- Assumes credentials that appear near each other are related

## HTML Processing

For HTML pages, SupaScan performs a comprehensive script analysis:

### Inline Scripts

```typescript
const INLINE_SCRIPT_PATTERN = /<script[^>]*>([\s\S]*?)<\/script>/gi;
```

All inline `<script>` tags are extracted and analyzed for credentials.

### External Scripts

```typescript
const SCRIPT_SRC_PATTERN = /<script[^>]+src=["']([^"']+)["']/gi;
```

SupaScan identifies all external script references and fetches them for analysis. Script URLs are resolved relative to the base URL:

```typescript
function resolveUrl(url: string, baseUrl: string): string {
  if (url.startsWith("http://") || url.startsWith("https://")) {
    return url;
  }

  const base = new URL(baseUrl);

  if (url.startsWith("//")) {
    return `${base.protocol}${url}`;
  }

  if (url.startsWith("/")) {
    return `${base.origin}${url}`;
  }

  const basePath = base.pathname.substring(
    0,
    base.pathname.lastIndexOf("/") + 1,
  );
  return `${base.origin}${basePath}${url}`;
}
```

## Security Implications

<Warning>
SupaScan can only extract credentials that are exposed in client-side code. If your application properly protects credentials using environment variables or server-side code, SupaScan won't be able to discover them - which is exactly what you want!
</Warning>

## What Gets Extracted

Successful extraction returns an `ExtractedCredentials` object:

```typescript
type ExtractedCredentials = {
  url: string;      // The Supabase project URL
  key: string;      // The anonymous/public API key
  source?: string;  // Where credentials were found
};
```

The `source` field helps you understand where the credentials were discovered:
- `"inline script"` - Found in an inline `<script>` tag
- A URL - Found in an external JavaScript file
- `undefined` - Found in the main content
