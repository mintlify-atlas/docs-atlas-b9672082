## What is SupaScan?

SupaScan is an automated security scanner for Supabase databases. Point it at any website using Supabase and it will extract credentials, discover database schemas, test Row Level Security (RLS) policies, and identify exposed data.

<Note>
SupaScan uses only the publicly available anon key to perform security analysis - the same credentials exposed to every visitor of a website.
</Note>

## How It Works

SupaScan operates in three phases:

<Steps>
  <Step title="Extract Credentials">
    Fetches the target website, parses inline scripts and external JavaScript bundles, and extracts the Supabase URL and anon key using pattern matching.
  </Step>
  
  <Step title="Analyze Database">
    Connects to the Supabase instance using the discovered credentials and systematically discovers:
    - Available schemas (public, auth, etc.)
    - Tables and their columns
    - RPC functions and their parameters
    - Row counts for accessible tables
  </Step>
  
  <Step title="Generate Report">
    Produces a detailed security report showing:
    - Exposed tables with row counts
    - Empty or RLS-protected tables
    - Access-denied tables
    - Callable RPC functions
  </Step>
</Steps>

## Use Cases

### Security Auditing

Identify misconfigured RLS policies before attackers do. SupaScan shows exactly what data is accessible with the public anon key.

### Penetration Testing

Perform authorized security assessments of Supabase-powered applications to find data exposure vulnerabilities.

### Development Validation

Verify that your RLS policies are working as intended during development and before production deployment.

<Warning>
SupaScan should only be used for authorized security testing. Always obtain permission before scanning databases you don't own.
</Warning>
